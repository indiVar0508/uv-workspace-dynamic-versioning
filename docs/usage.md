# Usage

## Basic Setup

To enable dynamic versioning in your `hatch` project, configure your `pyproject.toml` as follows:

```toml
[project]
name = "my-package"
dynamic = ["version"]

[tool.hatch.version]
source = "uv-workspace-dynamic-versioning"

[build-system]
requires = ["hatchling", "uv-workspace-dynamic-versioning"]
build-backend = "hatchling.build"
```

## Workspace Versioning

When using this plugin within a `uv` workspace, it automatically detects changes specific to the directory where the `pyproject.toml` resides. 

If you have a tag `v0.1.0` and make a commit in `packages/pkg-a`, only `pkg-a`'s version will reflect the distance (e.g., `0.1.0.post1+g<hash>`), provided other packages in the workspace didn't change.

## Dynamic Dependencies

You can also use this plugin to dynamically set your project dependencies. This is useful for referencing other packages in the same workspace that share the same versioning scheme.

```toml
[project]
name = "my-package"
dynamic = ["version", "dependencies"]

[tool.hatch.metadata.hooks.uv-workspace-dynamic-versioning]
dependencies = [
    "other-pkg == {{ version.base }}",
    "requests >= 2.28.0"
]
```

In this example, `{{ version.base }}` will be replaced by the base version (e.g., `0.1.0`) generated by the version source.

## Tag Patterns and "0.0.0" Issues

If your tags don't start with `v` (e.g., `1.0.0` instead of `v1.0.0`), the default `dunamai` pattern might not pick them up, resulting in a version like `0.0.0.postX`.

To fix this, configure the `pattern` in your `pyproject.toml`:

```toml
[tool.uv-workspace-dynamic-versioning]
# Match tags like '1.0.0' or 'release-1.0.0'
pattern = '(?P<base>\d+\.\d+\.\d+)'
```

The pattern must contain a named capture group `(?P<base>...)` for the base version.

## Hybrid Workspaces (Root + Members)

In a workspace, you might want the root project to use a standard dynamic versioning plugin (like `uv-dynamic-versioning`) while individual members use `uv-workspace-dynamic-versioning` to track their own directory-specific history.

Root `pyproject.toml`:
```toml
[tool.hatch.version]
source = "uv-dynamic-versioning"
```

Workspace Member `packages/pkg-a/pyproject.toml`:
```toml
[tool.hatch.version]
source = "uv-workspace-dynamic-versioning"
```

This setup ensures that:
1. The root package reflects the overall project version.
2. Each workspace member has its own version that reflects changes only within its own directory.
3. If no changes happened in a member since the last tag, its version will remain the same as the tagged version, even if the root version has advanced due to changes elsewhere.
